# Дизайн и юзабилити — Ежедневник

## Принципы

- **Read-only** — нет форм редактирования в UI
- **Быстрое чтение** — сканирование за 10 секунд
- **Не CRM** — проще: сегодня, календарь, проекты, карточка
- **Мобильно-дружелюбно** — адаптив с первого дня

## Экраны MVP

### 1. Сегодня (главный)
- Блок «Сейчас / сегодня»: встречи с временем
- Блок «Напоминания» без времени
- Блок «Задачи»: высокий приоритет, просроченные
- Цветовая полоса проекта слева (3D = синий, PROVANCE = зелёный)

### 2. Календарь
- Неделя по умолчанию, переключение на месяц
- События — карточки с клиентом и типом
- Конфликты подсвечивать красной рамкой (если есть в данных)

### 3. Проекты
- Таблица/карточки: имя, статус, последнее обновление
- Фильтры: тёплый, действующий, потенциальный, лид, партнёр

### 4. Карточка клиента
- Паспорт, статус сделки, backlog-таблица
- Журнал переговоров, открытые вопросы
- Ссылка на исходный `BACKLOG.md`

## Компоненты

| Компонент | Назначение |
| --- | --- |
| `ProjectBadge` | Цвет + название проекта (3D, PROVANCE) |
| `StatusChip` | Тёплый / Действующий / Просрочено |
| `EventCard` | Дата, время, клиент, тема |
| `TaskRow` | ID, задача, приоритет, статус |

## Типографика (рекомендация)

- Шрифт: system-ui или Inter
- Заголовки: 18–24px, тело: 14–16px
- Таблицы: zebra rows, sticky header
