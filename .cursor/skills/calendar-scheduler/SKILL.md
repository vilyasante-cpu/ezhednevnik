# Календарь — без пересечений

## Файлы

- **Главный календарь:** `3D/Проекты/КАЛЕНДАРЬ.md`
- **Клиент:** `3D/Проекты/[Имя клиента]/BACKLOG.md`

## Перед добавлением события

1. Прочитать блок **Предстоящие события** в `КАЛЕНДАРЬ.md`
2. Проверить конфликты:
   - **Встреча/звонок с временем** — не пересекается с другой встречей в тот же день и слот (±1 час буфер по умолчанию)
   - **Напоминание без времени** — не конфликтует
   - **Дедлайн** — может совпадать с встречами

3. При конфликте — сообщить пользователю и предложить 2–3 свободных слота

## Формат записи

```md
| ДД.ММ.ГГГГ | ЧЧ:ММ или — | **Клиент** | Встреча / Звонок / Напоминание / Дедлайн | Тема | **Статус** | Комментарий |
```

Статусы: `🔔 Запланировано`, `Подтверждено`, `Выполнено`

## После проведения

1. Перенести строку в **Прошедшие события** с итогом
2. Обновить **Журнал переговоров** в `BACKLOG.md` клиента
3. Обновить дату «Последнее обновление» в шапке файлов

## Шаблон в журнале backlog

```md
| ДД.ММ.ГГГГ | Встреча / Звонок | Участники | Итоги | Следующие шаги |
```

## Контрольные даты

Дедлайны без встречи — в блок **Контрольные даты** того же `КАЛЕНДАРЬ.md`.

## Синхронизация

После изменений напомнить: `python scripts/sync_data.py` (или кодер обновит в CI).
