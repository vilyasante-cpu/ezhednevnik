# Ежедневник — описание проекта

> MVP-продукт для единого обзора задач, встреч, напоминаний и заметок по всем проектам из корневой папки `CURSOR`.

**Дата фиксации:** 10.07.2026

---

## Суть

| Принцип | Описание |
| --- | --- |
| **Источник правды** | Markdown-файлы в проектах (`BACKLOG.md`, `КАЛЕНДАРЬ.md` и аналоги) |
| **Git / GitHub** | Только код и обезличенная статистика — см. [GIT-PRIVACY.md](GIT-PRIVACY.md) |
| **Режим работы** | Ежедневник **только показывает** данные. Все изменения — в чатах Cursor, агенты обновляют исходные файлы |
| **Охват** | Вся папка `CURSOR`: `3D`, `PROVANCE` и будущие проекты |
| **Цель UX** | Быстро читать планы: рабочая область, комментарии, напоминания, календарь, заметки — упрощённый CRM |

## Архитектура MVP

```
CURSOR/                          ← исходные markdown-файлы (редактируются агентами)
  3D/Проекты/КАЛЕНДАРЬ.md
  3D/Проекты/*/BACKLOG.md
  PROVANCE/...

Ежедневник/
  scripts/sync_data.py           ← сканирует CURSOR, собирает JSON
  data/planner.json              ← единый снимок для UI
  web/                           ← статический read-only интерфейс
  .cursor/skills/                ← субагенты (роли)
```

**Поток данных:**

1. Вы пишете в чат: «поставь встречу с BOXX на 15.07 в 14:00»
2. Агент-календарь проверяет пересечения → обновляет `КАЛЕНДАРЬ.md` и `BACKLOG.md`
3. `sync_data.py` пересобирает `planner.json`
4. Веб-интерфейс показывает актуальное состояние

## Экраны MVP (черновик)

1. **Сегодня** — задачи и напоминания на день
2. **Календарь** — неделя/месяц, цвет по проекту
3. **Проекты** — список с фильтром по статусу (тёплый, действующий, лид)
4. **Карточка клиента** — backlog, переговоры, открытые вопросы
5. **Заметки** — быстрые записи (позже)

## Облако (этап 2)

| Вариант | Плюсы | Минусы |
| --- | --- | --- |
| **GitHub Pages** | Бесплатно, git push = обновление | Нужен публичный или private repo |
| **Vercel / Netlify** | Автодеплой, HTTPS | Настройка CI |
| **Cursor Cloud Agent** | Синхронизация через git | Только просмотр после push |

Рекомендация для старта: статический сайт + автосборка `planner.json` при push в git.
