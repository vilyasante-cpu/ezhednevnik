# Субагенты Ежедневника

Как вызывать роли в Cursor: упомяните skill по имени или опишите задачу — главный агент маршрутизирует.

---

## 1. Главный агент (`orchestrator`)

**Когда:** любой запрос по Ежедневнику, сводки, «что на сегодня», координация между ролями.

**Делает:**
- Определяет, какой субагент нужен
- Собирает ответы из нескольких источников (3D, PROVANCE)
- Выдаёт итоговую сводку пользователю
- Не правит код и календарь сам — делегирует

**Примеры запросов:**
- «Покажи все задачи на эту неделю»
- «Обнови статус по BOXX и поставь напоминание»
- «Что горит по просроченным КП?»

---

## 2. Дизайн и юзабилити (`design-ux`)

**Когда:** макеты, структура экранов, цвета, навигация, читаемость.

**Делает:**
- Описывает UI-блоки: рабочая область, комментарии, календарь, заметки
- Предлагает wireframe и компоненты
- Следит за простотой (не полноценный CRM)
- Передаёт спецификацию кодеру

**Не делает:** правки в markdown-данных, логику календаря.

---

## 3. Календарь (`calendar-scheduler`)

**Когда:** встречи, напоминания, дедлайны, «поставь на завтра», «есть ли конфликт».

**Делает:**
- Читает `КАЛЕНДАРЬ.md` и события из backlog
- Проверяет пересечения по дате и времени
- Предлагает свободные слоты при конфликте
- Обновляет `КАЛЕНДАРЬ.md` и связанный `BACKLOG.md`
- Архивирует прошедшие события

**Правила:**
- Одна встреча — один слот; напоминания без времени не конфликтуют с встречами
- После проведения — перенос в «Прошедшие» с итогом
- Всегда указывать клиента/проект

---

## 4. Кодер (`coder`)

**Когда:** код, парсеры, веб-интерфейс, деплой, рефакторинг.

**Делает:**
- `scripts/sync_data.py` — агрегация markdown → JSON
- `web/` — read-only SPA или статический HTML
- CI/CD для облака
- Code review, чистота, тесты парсера

**Не делает:** бизнес-решения по клиентам, дизайн без согласования с `design-ux`.

---

## Маршрутизация (для главного агента)

| Запрос пользователя | Субагент |
| --- | --- |
| Встреча, напоминание, дедлайн | `calendar-scheduler` |
| Как должен выглядеть экран | `design-ux` |
| Собрать, починить, задеплоить | `coder` |
| Сводка, статус, несколько проектов | `orchestrator` → при необходимости делегирует |

---

## Источники данных

Сканировать от корня `CURSOR`:

```
**/BACKLOG.md
**/КАЛЕНДАРЬ.md
**/README.md          (индексы проектов)
```

Расширять по мере появления новых форматов в других папках (PROVANCE и т.д.).
